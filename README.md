# kitchen

Набор рабочих утилит, workflow и вспомогательных скиллов для команды Кожаных мешков: https://meatbags.ru

## Инструменты

| Раздел | Инструмент | Описание |
|---|---|---|
| Alfred | [Linear Issue Maker](./alfred/linear-issue-maker/) | Создание задач в Linear из Alfred |
| Python | [Wordstat Top Requests CLI](./python/wordstat/) | Запрос `topRequests` из API Яндекс.Wordstat |
| Skills | `sdelai-tasku` (`./.skills/sdelai-tasku/SKILL.md`) | Подготовка детального брифа задачи перед реализацией |

## Пользовательские сценарии

| Сценарий | Что использовать | Результат |
|---|---|---|
| Нужно быстро завести задачу без ручного заполнения полей в UI Linear | `alfred/linear-issue-maker` | Issue создается из Alfred с параметрами по умолчанию (team/project/assignee/priority) |
| Нужно собрать частотность по нескольким поисковым фразам для гипотез/контента | `python/wordstat` | Получаете JSON-ответы Wordstat для списка фраз, пригодные для последующей обработки |
| Нужно подготовить технически точный бриф задачи перед реализацией | `.skills/sdelai-tasku` | Структурированный task brief: требования из спеки, план реализации, риски, ожидания по бизнес-эффекту |

## Как работать с инструментами

### Alfred: Linear Issue Maker

1. Установите `Linear Issue Maker.alfredworkflow` из `alfred/linear-issue-maker/`.
2. В Alfred задайте минимум: `API_KEY`, `TEAM_ID`.
3. Вызовите ключевое слово workflow (по умолчанию `li`) и напишите заголовок задачи, например `li Fix auth bug`.
4. При необходимости используйте приоритет/флаги из локального `README`: `alfred/linear-issue-maker/README.md`.

### Python: Wordstat Top Requests CLI

1. Перейдите в `python/wordstat`.
2. Создайте `.env` на основе `.env.example` и заполните `WORDSTAT_API_KEY`.
3. Установите зависимости: `pip install -r requirements.txt` (рекомендуется в `.venv`).
4. Запустите скрипт: `python wordstat_top_requests.py "фраза 1" "фраза 2"`.

## Как пользоваться скиллом `sdelai-tasku`

Скилл нужен, когда перед кодингом требуется собрать полный бриф задачи на основе артефактов проекта.

1. Убедитесь, что в проекте есть 4 обязательных артефакта:
`dev-plan`, `project/feature spec`, `arch-patterns`, `task file`.
2. Дайте агенту команду вида:
`сделай задачу N`, `сделай PREFIX-N` или `сделай следующую задачу`.
3. Получите бриф с текущим статусом задачи, требованиями из спеки, техническим планом и рисками.
4. Только после подтверждения `делай` переходите к реализации.

Что нужно для работы скилла:
- Репозиторий со структурой `specs/` и `tasks/` (или эквивалентной, если определяется через `dev-plan`).
- Корректно заполненный `dev-plan` с `task_prefix` и `tasks_dir`.
- Актуальные файлы спецификации и архитектурных правил.
